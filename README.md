# Milling Machine Failure Prediction

Бинарная и multi-output классификация отказов фрезерного станка. Сравнение XGBoost, LightGBM, CatBoost.

**Ссылка на проект:** [Google Colab](https://colab.research.google.com/drive/111n9DRWltk2IylsfF2_tcKCWULcQ3mTI#scrollTo=P7v5xFgFNplt)

---

## Технологии
Python, Pandas, Scikit-learn, XGBoost, LightGBM, CatBoost, Matplotlib, Seaborn

---

## Этапы работы

### 1. Загрузка и анализ данных
- Датасет: 10 000 записей, 14 признаков
- 5 типов отказов: TWF (износ инструмента), HDF (проблема с охлаждением), PWF (проблема с мощностью), OSF (перегрузка), RNF (случайный отказ)
- Общий флаг Machine failure = 1, если хотя бы один тип отказа произошёл
- Сильный дисбаланс: отказы составляют ~3.5% от всех данных
- RNF: 18 примеров (0.18%), TWF: 51 пример (0.51%)

### 2. Feature Engineering
Созданы новые признаки на основе физических закономерностей из описания датасета:
- `temp_diff` = Process temperature - Air temperature (для HDF: отказ при temp_diff < 8.6K и speed < 1380 rpm)
- `power_approx` = Torque * Rotational speed (для PWF: отказ при мощности < 3500W или > 9000W)
- `wear_torque_product` = Tool wear * Torque (для OSF: отказ при превышении порога 11000-13000 в зависимости от типа)
- `osf_threshold` и `osf_exceed` - пороговые признаки для OSF
- One-hot encoding для категориального признака Type (L/M/H)

### 3. Моделирование
- **Бинарная классификация** (Machine failure): обучены XGBoost, LightGBM, CatBoost с подбором гиперпараметров через GridSearchCV (3-fold, scoring='roc_auc')
- **Multi-output классификация** (5 типов отказов): MultiOutputClassifier + XGBoost (обучена одна модель для всех 5 типов)

## Результаты бинарной классификации

| Модель | Precision (класс 1) | Recall (класс 1) | F1 (класс 1) | Accuracy |
|--------|---------------------|------------------|--------------|----------|
| XGBoost | 0.69 | 0.87 | 0.77 | 0.98 |
| LightGBM | 0.44 | 0.90 | 0.59 | 0.96 |
| CatBoost | 0.36 | 0.94 | 0.52 | 0.94 |

**Лучшие параметры XGBoost:**
- `colsample_bytree = 0.9`
- `learning_rate = 0.01`
- `max_depth = 7`
- `n_estimators = 300`
- `scale_pos_weight = 28.5`
- `subsample = 0.8`

## Результаты multi-output классификации (XGBoost)

| Тип отказа | Precision | Recall | F1 | Кол-во в test |
|------------|-----------|--------|-----|---------------|
| OSF | 1.00 | 1.00 | 1.00 | 16 |
| HDF | 0.93 | 0.97 | 0.95 | 29 |
| PWF | 0.86 | 0.92 | 0.89 | 13 |
| TWF | 0.00 | 0.00 | 0.00 | 10 |
| RNF | 0.00 | 0.00 | 0.00 | 4 |

**Почему OSF и HDF предсказываются идеально:** в датасете эти отказы генерируются по чётким физическим правилам (пороговые значения), которые модель легко выучила.

**Почему TWF и RNF не предсказываются:** TWF имеет всего 51 пример во всём датасете (10 в test) - модель не видит достаточно данных. RNF - случайный отказ с вероятностью 0.1%, он не зависит от признаков процесса, поэтому предсказание невозможно в принципе.

## Выводы

1. **XGBoost показал лучший результат для бинарной классификации** (F1=0.77) за счёт правильной настройки scale_pos_weight для борьбы с дисбалансом.

2. **OSF и HDF предсказываются идеально (F1=1.0 и 0.95)** благодаря чётким пороговым правилам в данных.

3. **TWF и RNF предсказать невозможно** из-за экстремального дисбаланса (51 и 18 примеров). RNF к тому же является случайным отказом и по определению не зависит от признаков.

4. **Feature engineering оправдан:** созданные физические признаки вошли в топ важности и улучшили качество моделей.

5. **Рекомендации:** для улучшения TWF необходимо собрать больше данных. RNF следует исключить из целевых переменных, так как он принципиально непредсказуем.
